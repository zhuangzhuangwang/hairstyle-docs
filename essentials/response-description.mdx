# AI发型设计 - 响应说明

## 公共响应字段

| 字段             | 必填     | 类型      | 范围                      | 描述                                                                     |
|:----------------|:---------|:----------|:---------------------------|:----------------------------------------------------------------------|
| `request_id`    | 是       | `string`  |                            | 请求ID。 <li>如果出现问题无法定位，可以将此字段发送给技术支持进行问题定位。</li>  |
| `log_id`        | 是       | `string`  |                            | 日志ID。 <li>如果出现问题无法定位，可以将此字段发送给技术支持进行问题定位。</li>  |
| `error_code`    | 是       | `integer` |                            | 状态码。 <li>当HTTP状态为200时：error_code=0；否则与HTTP状态完全相同。</li>   |
| `error_detail`  | 是       | `object`  |                            | 错误详情。                                                              |
| +`status_code`  | 是       | `integer` |                            | 状态码。                                                                |
| +`code`         | 是       | `string`  | [查看](#error-detail-code)  | 错误代码。                                                              |
| +`code_message` | 是       | `string`  |                            | 错误代码(`code`)表示的信息。                                              |
| +`message`      | 是       | `string`  |                            | `code_message`字段的补充信息。                                           |

## 如何验证请求是否成功？

<Warning>

- 使用HTTP状态码：
  - **200**: 成功。
  - **其他**: 失败。
- 使用`error_code`字段：
  - **0**: 成功。
  - **其他**: 失败。

</Warning>

<Card title="成功响应示例">
```json
{
  "request_id":   "请求ID",
  "log_id":       "日志ID",
  "error_code":   0,
  "error_detail": {
    "status_code":  200,
    "code":         "",
    "code_message": "",
    "message":      ""
  }
}
```
</Card>

<Card title="失败响应示例">
```json
{
  "request_id":   "请求ID",
  "log_id":       "日志ID",
  "error_code":   422,
  "error_detail": {
    "status_code":  422,
    "code":         "ERROR_NO_FACE_IN_FILE",
    "code_message": "文件中未检测到人脸。",
    "message":      "图片中没有人脸"
  }
}
```
</Card>
