# 响应字段说明

## 公共响应字段

| 字段 | 必填 | 类型 | 描述 |
|:-----|:-----|:-----|:-----|
| `request_id` | 是 | `string` | 请求ID。如果出现问题无法定位，可以将此字段发送给技术支持进行问题定位。 |
| `log_id` | 是 | `string` | 日志ID。如果出现问题无法定位，可以将此字段发送给技术支持进行问题定位。 |
| `error_code` | 是 | `integer` | 状态码。当HTTP状态为200时：error_code=0；否则与HTTP状态完全相同。 |
| `error_detail` | 是 | `object` | 错误详情。 |
| +`status_code` | 是 | `integer` | 状态码。 |
| +`code` | 是 | `string` | 错误代码。 |
| +`code_message` | 是 | `string` | 错误代码表示的信息。 |
| +`message` | 是 | `string` | `code_message`字段的补充信息。 |

## 如何验证请求是否成功？

<Warning>
- 使用HTTP状态码：
  - **200**: 成功
  - **其他**: 失败
- 使用`error_code`字段：
  - **0**: 成功
  - **其他**: 失败
</Warning>

## 成功响应示例

```json
{
  "request_id": "Request ID",
  "log_id": "Log ID",
  "error_code": 0,
  "error_detail": {
    "status_code": 200,
    "code": "",
    "code_message": "",
    "message": ""
  }
}
```

## 失败响应示例

```json
{
  "request_id": "Request ID",
  "log_id": "Log ID",
  "error_code": 422,
  "error_detail": {
    "status_code": 422,
    "code": "ERROR_NO_FACE_IN_FILE",
    "code_message": "No face detected in the file.",
    "message": "pic not has face"
  }
}
```
